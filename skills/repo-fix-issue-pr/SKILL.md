# Fix issue PR workflow

Work on one approved issue at a time.

1. Inspect the issue and existing open pull requests with `gh`.
2. Stop if another contributor already has relevant work open.
3. Use an isolated worktree and an `ai/fix` branch generated by the orchestrator.
4. Reproduce or characterize the issue before editing when practical.
5. Make the smallest safe change.
6. Run targeted checks.
7. Open a pull request only when explicitly requested by the task.

Never merge, force push, delete branches, expose secrets, or take over another
author's work.
