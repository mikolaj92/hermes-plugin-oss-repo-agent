Reviewer retry proposal deliberately rejected: the approved migration contract requires Fala-native default max_attempts=1 for every process. Fala 0.7.6's package host does not expose per-effector max-attempt overrides through its Python request, and its subprocess adapter maps nonzero exits to adapter failures without the structured Result classification. Raising attempts would therefore retry terminal and possibly mutating failures indiscriminately. Keeping one attempt is the only fail-closed policy supported by the pinned runtime and matches the plan acceptance criterion.
